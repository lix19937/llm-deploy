

## AI Agent 的 Skill 与 Tool：为什么需要Skill层？  

在构建 AI Agent 系统时，我们通常关注两个核心问题：Agent 如何理解指令，以及 Agent 如何执行动作。  
前者涉及 Prompt 工程，后者涉及工具（Tool）调用。但随着 Agent 应用场景的复杂化，Skill（技能） 已经成为 Agent 的标配。

## 从 Tool 说起：Agent 的原子能力       
Tool 是什么？Tool（工具）是 Agent 与外部世界交互的最小单元。每个 Tool 执行一个原子操作，比如：     
```
search_web(query)       —— 搜索网页   
read_file(path)         —— 读取文件   
run_command(cmd)        —— 执行命令行    
send_email(to, content) —— 发送邮件      
```
Tool 的特点是低层次、高灵活性、无状态。它只负责“做好一件事”，不关心业务场景和上下文。

## Tool 的局限      
让我们看一个场景：让 Agent 完成“代码审查”。
```
如果只用 Tool，Agent 需要自己编排：     
1. 调用 read_file 读取源代码   
2. 调用 grep 搜索潜在问题   
3. 调用 run_command 执行 linter    
4. 调用 write_file 生成审查报告     
5. 自己处理每个步骤的错误    
6. 自己决定输出格式   
```

这意味着 Agent 需要：

理解代码审查有哪些步骤    
知道用什么工具组合  
处理各种边界情况（文件不存在、linter 报错等）    
自行设计输出格式     

问题在于：这些细节与“代码审查”这个业务目标无关，Agent 需要花费大量 token 来理解这些低层次操作。  

## Skill 的诞生：封装业务语义
Skill 是什么？Skill（技能）是一组 Tool 的编排组合，它携带了更丰富的语义信息和约束规则。    

类比理解：       
```
Tool  = 螺丝刀、锤子、钉子（单个工具）
Skill = “组装椅子”这个技能（工具 + 步骤 + 约束） 
```

### Skill 的特点
|特点 |	说明 |    
| ---| ---- |    
|组合性	|多个 Tool 的编排，按顺序执行|    
|有状态	|携带上下文信息，上一步输出作为下一步输入|     
|高层次	|封装业务语义，Agent 只需理解“做什么”|      
|可约束	|可附带 Harness 规则，限制行为范围|

### 对比示例
|维度 |	Tool	|Skill|     
| ---| ----  |----|    
|粒度	   |原子操作|	业务能力|        
|语义	   | 低层次（“读文件”）	|高层次（“代码审查”）|       
|使用方式	|直接调用	|按场景调用|
|错误处理	|Agent 自行负责	|Skill 内部处理|
|约束	|无	|可带 Harness| 

## 为什么需要 Skill？
### 核心价值   
1. 降低 Agent 复杂度     
使用 Skill 后，Agent 只需说“帮我审查这个 PR”，Skill 内部会自动：
```
读取代码
运行 linter
分析潜在问题
生成格式化的审查报告
```
Agent 不需要知道具体用什么工具、如何编排。

2. 提高可靠性   
>Skill 的开发者已经处理了各种边界情况：
> 文件不存在怎么办？
> linter 运行失败怎么办？
> 超时怎么处理？
 
这些细节被封装在 Skill 内部，调用者无需关心。

3. 可复用性与可审计性   
> 同一个 Skill 可以被多个 Agent 复用：  
“代码审查” Skill 可以用于 PR 审查、代码合并前检查、代码质量监控等场景
每次 Skill 调用都有记录，方便审计追踪。

4. 携带业务约束（Harness）  
> Skill 可以附带行为约束，例如：
“删除数据” Skill：必须确认、必须记录日志、禁止批量删除
“发布部署” Skill：需要审批、必须经过测试

这是 Tool 无法做到的。

## 总结     
|问题 |	回答 |  
|----|-----|     
|Tool 是什么？|	Agent 调用外部能力的原子操作|    
|Skill 是什么？|	一组 Tool 的编排 + 业务语义 + 约束|    
|为什么需要 Skill？|	降低 Agent 复杂度、提高可靠性、携带业务约束|   

----

**Tool 是“工具”，Skill 是“技能包”。用技能包比用原始工具更高效、更可控。**
